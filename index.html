Quick graph key

Fast reminder only. Full rules stay in the Guide.
Container: quantity held by an owner.label owner · unit thing · attribute subtype/state.
Transfer: before/after change.
Directionbefore → after
Equationafter = before ± quantity
Rate: per-unit or grouping relation.
Directionnumerator → denominator
Equationnumerator = rate × denominator
Difference: additive comparison.
Directionfewer side → more side
Equationmore = fewer + difference
Explicit: multiplicative scaling.
Directionbase → scaled
Equationscaled = base × factor
Part: parts build a whole.
Directionpart → whole
Equationwhole = sum(parts)
ref: question target.The node the question asks for.

Quick label order
You may select multiple labels when the graph has more than one independent problem, but choose the main root cause first.
1. Nodes first. Missing or unsupported quantity box → Missing node / Extra node.
2. Relations next. Missing or unsupported edge → Missing relation / Extra relation.
3. Then ref. Use Wrong reference target only when the correct answer node already exists.
4. Then edge setup. Same endpoints but wrong relation type → Wrong relation type. Same relation type but arrow flipped → Wrong direction.
5. Content last. Wrong number or variable → Wrong quantity. Wrong owner, unit, or attribute → Wrong node or relation content.
Treat missing/extra structure as the main cause. Do not jump to relation type, direction, or content if an earlier structural problem already explains the graph.
